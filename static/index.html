    api_key="fa_sk_your_api_key",

    model="gemini-2.5-flash",
    messages=[

    ]
)
print(response.choices[0].message.content)

Подкапотная Память

ИИ может сохранять данные между сессиями через специальные теги в ответе. Теги невидимы пользователю — бэкенд вырезает их перед отправкой.

context.md
Сжатый конспект сессии на английском. Перезаписывается при каждом обновлении. Восстанавливается после /reset.
favorite.md
Постоянные настройки: роль, стиль общения, инфо о пользователе. Сохраняется навсегда, не сбрасывается при /reset.
ТегОписаниеПравило
【⊕write:ctx⊕】...【⊕/write:ctx⊕】Записать context.mdТолько EN, в конце ответа
【⊕write:fav⊕】...【⊕/write:fav⊕】Записать favorite.mdПерсона и роль, хранится вечно
【⊕load:mem⊕】Запросить загрузку памятиБэкенд инжектирует контент в следующий запрос

Dual-Account (Экономия токенов)

Один ключ — два Telegram-аккаунта: основной (запросы к ИИ) и переводчик (RU→EN до запроса, EN→RU ответа).

Экономия
1 слово EN ≈ 1.3 токена
1 слово RU ≈ 3.3 токена
Dual-режим экономит до 2.5x (≈60%) токенов при переводе RU→EN
Включить: Настройки ключа → переключатель Dual-Account → выбрать аккаунт-переводчик.

Системный промпт для внешних ИИ

Вставьте этот промпт в системный промпт внешнего ИИ-ассистента, чтобы он умел использовать FavoriteAPI и теги памяти.

SYSTEM PROMPT
# FavoriteAPI — System Prompt for External AI Integration

FavoriteAPI is a self-hosted proxy service that routes AI requests through
Telegram bots (currently using Gemini models via @SamGPTrobot).
There is no public domain yet — it runs locally or via Cloudflare tunnel.

## Making Requests
- Endpoint: POST /api/v1/chat
- Auth: Authorization: Bearer <api_key>  (format: fa_sk_<64hex>)
- Body: {"messages": [{"role": "user", "content": "..."}], "model": "gemini-2.5-flash"}
- Response: {"choices": [{"message": {"role": "assistant", "content": "..."}}],
             "context_kb": 12.4, "log_code": "REQ_OK_001"}

## Token Economy
- Writing in ENGLISH saves ~2.5x context tokens vs Russian/Cyrillic (1 RU word ≈ 3.3 tokens, 1 EN word ≈ 1.3 tokens).
- Context limit: ~180 KB. You will receive CTX_LIMIT_180 error when hit.
- context_kb field in every response shows current accumulated context.

## Memory System (Under-the-hood Tags)
You can persist memory across context resets using special tags.
Tags use 【⊕...⊕】 brackets — they are stripped from user-visible output.

WRITE context summary (compress conversation, English only):
  【⊕write:ctx⊕】
  Your compressed summary here. User language: ru.
  Key topics: ...
  【⊕/write:ctx⊕】

WRITE favorite settings (role, style, user info — persists forever):
  【⊕write:fav⊕】
  Role: senior developer assistant. Style: concise, technical.
  User: works on Python Flask API project.
  【⊕/write:fav⊕】

REQUEST memory load (backend injects memory in next request):
  【⊕load:mem⊕】

Rules:
- Tags must appear at the END of your response, after user-visible content.
- write:ctx should be in English only (1 EN word ≈ 1.3 tokens vs 3.3 for RU — saves ~60%).
- Use write:fav for persistent persona/role settings.
- Backend strips tags before sending response to user — they are invisible.
- When context nears limit, you will receive a SYSTEM NOTICE — act on it.

## Reset Flow
- POST /api/v1/reset — resets Telegram bot context
- If CTX_LIMIT_180 occurred, reset returns {requires_choice: true} with
  options to apply or clear your saved memory files.
- After restore, include 【⊕load:mem⊕】 in first response to reload context.

## Available Models
- gemini-2.5-flash (recommended, 200k context)
- gemini-2.5-pro, gemini-2.5-mini, gemini-2.0-flash (various sizes)

## Important
- One request at a time per API key (KEY_BUSY_301 if concurrent)
- Images: include as base64 data URLs or http URLs in content array
- Streaming: add "stream": true for SSE format

АУДИТ ПРОЕКТА
=================================================================
Стек: Flask + SQLite + Telethon, single-page HTML (~5300 строк)
Среда: Termux (Android), Cloudflare Tunnel для публичного URL
Особенность: использует rvXhr (XMLHttpRequest), а НЕ fetch()
  → fetch() может ненадёжно работать на Android WebView/Termux
  → Автор уже знал об этом и написал rvXhr специально

НАЙДЕННЫЕ БАГИ:
1. submitWriteReview() использует api() [fetch] — НЕ работает на Android
   → Исправление: переписать через rvXhr()
2. onWriteReviewFilesSelected() использует compressImage [canvas]
   → canvas.toDataURL() может падать на мобильном браузере
   → Исправление: FileReader.readAsDataURL() напрямую
3. writeReviewFileInput: был display:none + pointer-events:none
   → Исправление: <label> wrapping input
4. Вся api() функция использовала fetch() — ненадёжна на Android
   → Исправление: полный рефакторинг api() на XHR (Promise wrapper)
5. Все остальные fetch() вызовы — logout, loadReviews, auth/me, TG import
   → Исправление: все переведены на XHR

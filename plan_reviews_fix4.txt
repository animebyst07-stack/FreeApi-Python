ПЛАН: Исправление кнопки "Написать отзыв" в разделе отзывов
Дата: 2026-04-19

ПРОБЛЕМА:
Авторизованные пользователи (включая админа) не видели кнопку "Написать отзыв"
в разделе отзывов (/reviews).

ПРИЧИНА:
Функция initReviewsView() вызывается при переходе в раздел отзывов (goView('reviews')).
Она проверяет глобальную переменную `user`. Если пользователь открывал раздел отзывов
до завершения асинхронного запроса /api/auth/me (который инициализируется при загрузке
страницы), `user` оставался null. В результате показывалась кнопка "Войти" вместо
кнопки "Написать отзыв", даже для авторизованных пользователей.

ИСПРАВЛЕНИЕ (static/index.html):
1. Выделена отдельная функция _rvRenderWriteBar() для отрисовки блока написания отзыва.
   Она проверяет переменную user и показывает:
   - кнопку "Написать отзыв" → openWriteReviewModal() (для авторизованных)
   - блок "Войти" (для гостей)

2. Функция initReviewsView() переработана:
   - Если user уже загружен — сразу отрисовывает _rvRenderWriteBar()
   - Если user = null — делает XHR-запрос к /api/auth/me, обновляет user/accountState,
     затем вызывает _rvRenderWriteBar(). Это гарантирует корректное отображение даже
     при быстром переходе на страницу отзывов до завершения начальной инициализации.

РЕЗУЛЬТАТ:
Авторизованные пользователи (включая ReZero-админа) всегда видят кнопку
"✏ Написать отзыв" в разделе отзывов. Клик по ней открывает модальное окно
writeReviewModal с формой (функционал уже существовал — openWriteReviewModal).
Незарегистрированные пользователи по-прежнему видят блок "Войти".

ФАЙЛЫ ИЗМЕНЕНЫ:
- static/index.html (функция initReviewsView + новая _rvRenderWriteBar)

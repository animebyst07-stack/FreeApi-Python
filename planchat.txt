ЦЕЛЬ
────
Интерактивный тест-чат с ИИ прямо из дашборда. Пользователь выбирает
API-ключ, выбирает модель, пишет сообщения — получает ответы. Контекст
сохраняется в рамках сессии (массив messages[]). Все запросы логируются.


BACKEND: /api/chat/test
───────────────────────
POST /api/chat/test
  Body: { keyId: string, model: string, messages: [{role, content}] }
  Auth: требует сессию (require_user)
  Логика:
    1. get_user_key(user_id, keyId) — проверяем владение ключом
    2. если is_busy — 409
    3. run_chat(key, model, messages) — внутренний вызов (тот же, что /api/v1/chat)
    4. возвращает {answer, model, responseMs}
  Это позволяет не передавать Bearer-токен на фронтенд.

POST /api/chat/reset
  Body: { keyId: string }
  Auth: require_user
  Логика:
    1. get_user_key(user_id, keyId)
    2. run_control(key, '/reset')
    3. возвращает {ok: true}

ФОРМАТИРОВАНИЕ (Telegram/Markdown style)
─────────────────────────────────────────
Парсер на чистом JS (без библиотек):

Блоки (обрабатываются первыми):
  ```lang\ncode\n```  → <pre><code class="lang">...code...</code></pre>

Инлайн (после блоков):
  **text** или __text__  → <strong>text</strong>   (bold)
  *text* или _text_      → <em>text</em>            (italic)
  ~~text~~               → <del>text</del>          (strikethrough)
  `code`                 → <code>code</code>        (inline code)
  [text](url)            → <a href=url>text</a>     (link)

Строчные элементы:
  строка начинается с "> " → blockquote
  строка начинается с "# " → h1/h2/h3 по кол-ву #
  строка начинается с "- " или "* " → li (ul)
  строка начинается с "N. " → li (ol)
  пустая строка           → разрыв параграфа

XSS-защита: escapeHtml() перед добавлением в DOM.
После парсинга устанавливается через innerHTML.

БЛОК 0. БАГ С БАНОМ ОТЗЫВОВ (фикс по ходу запроса)
────────────────────────────────────────────────────────────────────────

Текущее поведение:
  • agent.py: одно удаление от модератора → ban на 7 дней.
  • reviews_bp.py: ban-проверка скипается, если username == 'ReZero', — ОК.
  • Фронт (reviews.js / writeReview-модалка) показывает «доступ ограничен»
    из общего флага, не различая админа. → Расхождение, которое ты видишь.

Желаемое поведение:
  • Скользящее окно 7 дней. Удалённый модератором отзыв = +1 к счётчику.
  • При достижении 5/5 за окно → бан на 7 дней (одна неделя без отзывов).
  • Окно автоматически "сбрасывается": считаем только записи моложе 7 дней.
  • Админ (ReZero) полностью исключён и из счётчика, и из бана, и из UI-плашки.
  • Пользователю в /api/auth/me и /api/reviews/mine отдаём
    { removals_window: { count, max: 5, reset_at, days_left }, ban: {...} }
    чтобы UI рисовал «3/5 за неделю» и точное время сброса.

Правила:
  • Всё через require_user() / is_admin() из _helpers.py.
  • Slow-mode: не-админу нельзя слать чаще 1 сообщение / 3 сек (in-memory rate limit).
  • Лимит длины текста 4000 симв., до 3 фото ≤200KB.
  • Парсер @login (regex r'@([a-zA-Z0-9_]{3,32})') → разрешает только реально
    существующих юзеров (lookup в users.username).
  • При создании mention’а → create_user_notification(kind='community',
    ref_id='msg:<id>'), если у адресата notif_mute_mentions=0.
  • Soft-delete: запись остаётся, image-blob остаётся; на API наружу
    отдаётся плашка «сообщение удалено · кем» (только текстовый стаб),
    оригинал доступен админу через /history.

            юзер видел текст без фото. По правилам Replit мы не
            храним base64-фото гигабайтами, но 2.5 MB на сообщение
            это разумный middle-ground.
            Фикс бэка (repos/community.py): MAX_IMG_BYTES = 2_500_000
            (2.5 MB после base64). С запасом на 1 фото в JPEG 100%.
            Фикс фронта (community.js): новые _CM_MAX_SIDE=1280 и
            _CM_JPEG_Q=0.85 + функция _compressDataUrl: каждое фото

            1280, toDataURL('image/jpeg', .85)). Это типичные
            настройки Telegram для inline-фото — снимок 4032×3024
            (3 MB) ужимается до 1280×960 (~150-300 KB). Если результат

            На любой ошибке (img.onerror, нет canvas-ctx) возвращаем
            оригинал — бэк теперь и так его примет. _readFilesAsDataUrls
            делает FileReader → _compressDataUrl последовательно для
            каждого файла, результаты собираются Promise.all.
            Использует и chat-composer (cmHandleFiles), и
            post-composer (cmHandlePostFiles) через ту же функцию.
            Файлы: freeapi/repos/community.py (MAX_IMG_BYTES),
            static/js/views/community.js (_compressDataUrl,

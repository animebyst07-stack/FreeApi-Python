нет кнопки начать новый диалог — нужна перезагрузка.

Решение (index.html):
- В HTML support-header добавить кнопку <button id="supportNewChatBtn">
  "Новый диалог" (изначально display:none)
- В _doCloseSupportChat() после успешного закрытия показать эту кнопку
- Добавить функцию startNewSupportDialog():
  * Очистить все сообщения из supportMessages (кроме supportEmptyState)
  * Показать supportEmptyState
  * Разблокировать supportInput и supportSendBtn
  * Скрыть supportNewChatBtn и supportCloseBtn
  * Сбросить supportChatStatus текст
  * Сбросить _supportPending = false
  * Очистить вложение (clearSupportAttachment)

ЗАДАЧА 3: Логин пользователя prominently в диалоге поддержки
-------------------------------------------------------------
Проблема: username передаётся только в системный промпт,
но не в каждое сообщение — ИИ может путать пользователей.

Решение (freeapi/routes.py):
- _format_support_dialog_for_ai(messages) → добавить параметр username
  Метку "Пользователь" заменить на "Пользователь (@username)"
  В финальной инструкции упоминать username явно
- В support_send_message() извлечь username и передать в функцию

ЗАДАЧА 4: Баг создания второго ключа (пустая страница)
------------------------------------------------------
Проблема: createKey() просто показывает setupSection (форма
подключения TG), но loadDashboard() её сразу скрывает если
есть ключи. Кнопка "Новый ключ" ничего не делает видимого.

Решение (index.html):
- Изменить createKey(): если accountState.has_tg_account — сразу
  вызвать POST /api/keys через API, показать rawKeyModal с ключом
  (как при setupCompleted) вместо показа setupSection
- Если нет TG-аккаунта — прежнее поведение (показать setupSection)
- Установить правильный заголовок rawKeyModal: "Ключ создан!"

ЗАДАЧА 5: Ширина селектора модели в чате слишком мала (170px)
-------------------------------------------------------------
Проблема: длинные имена моделей обрезаются в триггере,
в выпадающем списке тоже не видно полного имени.

Решение (index.html):
- chatModelSelectWrap: заменить width:170px на flex:1;min-width:180px;max-width:280px
- Добавить CSS для #chatModelDropdown .custom-select-option:
  white-space:normal; word-break:break-word; → полные имена видны в списке

ЗАДАЧА 6: Нативный <select> аккаунта-переводчика → кастомный
-------------------------------------------------------------
Проблема: <select id="ksDualTranslator"> не соответствует стилю
остальных кастомных дропдаунов (custom-select-wrap).

Решение (index.html):
- Заменить <select id="ksDualTranslator"> на:
  <div class="custom-select-wrap" id="ksDualTranslatorSelect">
    <div class="custom-select-trigger"> ... </div>
    <div class="custom-select-dropdown"></div>
    <input type="hidden" id="ksDualTranslatorVal" value="">
  </div>
- В openKeySettings: заменить прямую запись tsel.innerHTML=...
  на вызов buildCustomSelect('ksDualTranslatorSelect', opts, selectedVal)
- В saveKeySettings: читать document.getElementById('ksDualTranslatorVal').value

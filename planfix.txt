A. БЭКЕНД (Python / Flask + Telethon)
--------------------------------------

Что реализовано (работает):
  [+] Регистрация / вход / сессия через Flask session
  [+] Шифрование api_hash и session_string (XOR + HMAC-SHA256)
  [+] SQLite база данных с правильной схемой (users, tg_accounts,
      api_keys, requests, model_stats, log_codes, setup_sessions)
  [+] Настройка TG-аккаунта (SetupFlow): SpamBot → обучение →
      настройки GPT → промокоды → сброс
  [+] Обработка спонсоров (SponsorHandler) с rate limiting
  [+] Активация промокодов из канала (PromoActivator)
  [+] Смена модели через интерфейс бота (ModelSwitcher)
  [+] Отправка длинных запросов как .txt файл (>4096 символов)
  [+] Поддержка изображений (по одному, с паузами)
  [+] Фильтрация подсказок 💡 (skip_hints)
  [+] Блокировка параллельных запросов на ключ (threading.Lock)
  [+] /stop и /reset команды через API
  [+] Статистика моделей (avg_response_ms, total_requests)
  [+] Глобальная статистика (users, todayRequests, totalRequests)
  [+] Лог-коды в БД (30 кодов по категориям)
  [+] SSE стриминг прогресса настройки
  [+] Полный CRUD для API-ключей

Найденные проблемы бэкенда:
  [!] SponsorHandler.join_url — нет обработки ошибок (ИСПРАВЛЕНО)
  [!] PromoActivator.activate — падает на первом сломанном промокоде (ИСПРАВЛЕНО)
  [!] collect_answer: при BOT_QUIET_SECONDS=2.0 может собрать неполный
      ответ если бот пишет с задержкой более 2 сек между частями
  [!] run_chat использует asyncio.run() в threaded Flask — это создаёт
      новый event loop в каждом потоке. При большой нагрузке может
      накапливать незакрытые loop. Нужен ThreadPoolExecutor с
      loop per thread.
  [!] scheduler.py — заглушка, ничего не делает. Нужна ротация
      старых сессий и очистка зависших setup_sessions.
  [!] Нет валидации длины api_id (должен быть числом, 6-10 цифр)
  [!] После setup: setup_sessions с status='running' не очищаются
      если сервер перезапустился — при следующем запуске
      get_running_setup вернёт "зависшую" сессию. Нужен сброс
      при старте.
  [!] update_model_stats: при concurrent запросах race condition
      на SQLite (хотя RLock частично защищает)
  [!] /api/keys/<id> — возвращает key_value БЕЗ mask_key,
      а /api/keys — с mask_key. Несогласованность API.
  [!] Нет rate limiting на /api/auth/register — можно создать
      тысячи аккаунтов

VII. БАГ: SETUP_FAIL_604 — UserBlockedError при /start
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Симптом:
  Step 2 "Запускаем ИИ..." падает с ошибкой:
  "SETUP_FAIL_604: You blocked this user (caused by SendMessageRequest)"

Причина:
  Пользователь ранее заблокировал бота (@SAM_BOT_USERNAME) в Telegram.
  При send_message Telethon выбрасывает UserBlockedError.
  Текущий код не импортирует UserBlockedError и не обрабатывает его.

Фикс (freeapi/tg.py):
  1. Импортировать UserBlockedError из telethon.errors
  2. Импортировать UnblockRequest из telethon.tl.functions.contacts
  3. В методе TgSession.send_message — перехватывать UserBlockedError,
     вызывать UnblockRequest(entity), затем повторять send_message один раз
  4. Обновить сообщение об ошибке в SetupFlow.run() — различать
     UserBlockedError отдельно с понятным текстом
